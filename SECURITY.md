# Security Policy

## Supported Versions

| Version | Supported |
| ------- | --------- |
| 0.1.x   | ✅ Active development |

## Reporting a Vulnerability

We take the security of our platform seriously. If you believe you have found a security vulnerability, please report it to us as described below.

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, please report them via GitHub's **Security Advisory** feature:

1. Go to https://github.com/lawaladekunle/taiyeandkehindeai/security/advisories/new
2. Provide a detailed description of the vulnerability
3. Include steps to reproduce the issue
4. Note the potential impact

Alternatively, you can email security@taiyeandkehindeai.com.

You should receive a response within 48 hours. If you do not, please follow up to ensure we received your report.

## What to Expect

- **Acknowledgment** of your report within 2 business days
- **Validation** of the issue and determination of impact
- **A fix** targeting the next patch release for validated issues
- **Public disclosure** coordinated between us and you after the fix is released

## Scope

This security policy covers:
- The web application and API (backend + frontend)
- Infrastructure and deployment configurations
- CI/CD pipeline configurations
- Third-party dependencies

## HIPAA Compliance

This platform handles Protected Health Information (PHI). If you find a vulnerability that could expose PHI, please mark your report as **critical** and we will respond immediately.

## Preferred Languages

We prefer all communications to be in English.

## Recognition

We maintain a hall of fame for researchers who responsibly disclose security issues. With your permission, we will acknowledge your contribution.

## Disclosure Timeline

We aim to:
- **Triage**: Within 2 business days
- **Fix**: Within 14 days for high/critical issues
- **Disclosure**: Coordinated public disclosure after fix deployment
